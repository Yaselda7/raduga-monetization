# Градация клиентов

## Принцип
Разные клиенты требуют разных моделей работы, подключения ИИ и ценообразования.
Понять тип клиента — значит правильно выстроить продукт и модель оплаты.

---

## Типы клиентов

### Микро-клиент
**Кто:** маникюрша, репетитор, мастер, небольшой локальный сервис
**Характеристика:**
- Нет сайта или есть только страница в Instagram
- Нет технических знаний и нет желания их получать
- Нет времени разбираться
- Не готов к ежемесячным платежам нескольким сервисам
- Впадёт в ступор от API ключей и подписок на Claude/ChatGPT

**Модель работы:**
- Всё под ключ — ты берёшь на себя инфраструктуру
- Клиент платит тебе единую абонентку (включает ИИ + поддержку)
- Максимальная простота интерфейса — одна кнопка, один результат
- Ты арендуешь сервер/подписку, клиент не знает деталей

**Подключение ИИ:** через твою инфраструктуру (виртуальный сервер или твоя подписка)

**Продукты:** уровень 1-2, узкие, под одну задачу

---

### Малый клиент
**Кто:** небольшой бизнес, фрилансер, эксперт с аудиторией
**Характеристика:**
- Есть Instagram и/или сайт-визитка
- Базовое понимание цифровых инструментов
- Готов к одной подписке если видит ценность
- Хочет минимального обучения

**Модель работы:**
- Инструмент + краткое обучение (1-2 часа)
- Клиент сам управляет базовыми настройками
- Поддержка по запросу

**Подключение ИИ:** через твою инфраструктуру или клиент заводит свою подписку
с твоей помощью

**Продукты:** уровень 1-2, с небольшой кастомизацией

---

### Средний клиент
**Кто:** компания с сайтом, есть технический человек внутри
**Характеристика:**
- Работающий сайт с регулярными обновлениями
- Есть кто-то технически грамотный в команде
- Понимает ценность автоматизации
- Готов к интеграциям

**Модель работы:**
- Интеграция в существующие процессы
- Обучение команды
- Первоначальная поддержка, затем самостоятельность

**Подключение ИИ:** API ключи или собственная подписка клиента

**Продукты:** уровень 2-3, интеграции, автоматизации процессов

---

### Крупный клиент
**Кто:** большая компания, организация, команда с объёмными задачами
**Характеристика:**
- Большой объём задач по автоматизации
- Несколько отделов или направлений
- Нужны системные решения
- Есть бюджет и техническая команда

**Модель работы:**
- Системное решение под задачи бизнеса
- Проектный подход
- Документация и передача

**Подключение ИИ:** API ключи, корпоративные подписки

**Продукты:** уровень 3, мультиагентные системы, платформы

---

## Матрица: профиль → комфортный тип клиента

| Профиль | Комфортный клиент | Почему |
|---|---|---|
| 🟣 Архитектор | Средний, Крупный | Любит сложные задачи, не хочет объяснять простое |
| 💙 Аналитик | Средний, Крупный | Системное мышление, нужен масштаб |
| 🔵 Наставник | Малый, Средний | Любит обучать, нужен контакт |
| 🟢 Просветитель | Микро, Малый | Тепло, терпение, забота о каждом |
| 🟡 Мастер | Малый, Средний | Качество важнее масштаба |
| 🟠 Творец | Малый | Живой контакт, не хочет рутины поддержки |
| 🔴 Спринтер | Любой, кто купит быстро | Важна скорость сделки, не тип клиента |

---

## Важное наблюдение
Микро-клиентов больше всего — но с ними и проблем больше.
Для работы с ними нужна модель максимальной простоты со стороны клиента
и максимальной автономии продукта.

Это естественная ниша для Просветителя и Наставника —
они готовы объяснять и сопровождать.

Архитектору и Аналитику микро-клиент будет некомфортен —
слишком простые задачи, слишком много ручного объяснения.
